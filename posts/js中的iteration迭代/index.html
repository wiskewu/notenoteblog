JS中的Iteration迭代

2026/4/24 14:12:1149 分钟阅读
Javascript

前言

ES6引入了for...of循环语法，减少标准循环的复杂性和避免记录循环索引导致的索引越界问题。

let ranks = ['A', 'B', 'C'];

for(let rank of ranks) {
    console.log(rank);
}

for...of语法相比于for循环更加优雅一些，因为它更加明确循环的意图：迭代数组以访问序列中的每个元素。除此之外，for...of循环还可以迭代任何可迭代的对象，而不仅仅只是迭代一个数组。

要理解可迭代对象，首先需要理解迭代协议。

迭代协议

Note

根据MDN-迭代协议描述，迭代协议并不是新的内置实现或语法，而是协议。这些协议可以被任何遵循某些约定的对象来实现。 迭代协议具体分为两个协议：可迭代协议（iterable protocol）和迭代器协议（iterator protocol）。

在继续阅读下文时，有三个概念需要熟悉：

  • 可迭代的对象：一个可迭代的对象，是具有返回迭代器对象的特殊迭代器方法的任何对象；
  • 迭代器对象本身：迭代器是指任何具有实现next()方法，并返回迭代结果对象的对象；
  • 迭代结果：一个具备value和done属性的对象；

可迭代协议

要成为可迭代对象，该对象必须实现[Symbol.iterator]()方法。

  • 该方法不接收任何参数；
  • 该方法返回一个遵循迭代器协议的对象；
  • 此函数可以是普通函数，也可以是生成器函数（Generator），在生成器内部，可以使用yield提供每一项值；

Note

Symbol.iterator是 ES6 引入的内置symbols之一。

迭代器协议

迭代器协议定义了产生一系列值（无论是有限个还是无限个）的标准方式，当值为有限个时，所有的值都被迭代完毕后，则会返回一个默认返回值。

一个对象，只要实现具备以下语义的next()方法，就可以视为迭代器：

next()

无参数或者接受一个参数的函数。并返回符合InteratorResult接口的对象。

return(value)（可选的函数）

无参数或者接受一个参数的函数，其返回符合InteratorResult接口的对象。调用这个方法表明迭代器的调用者不打算调用更多的 next()，并且可以进行清理工作。

throw(exception)（可选的函数）

无参数或者接受一个参数的函数，其返回符合InteratorResult接口的对象。调用这个方法表明迭代器的调用者监测到错误的状况，并且 exception 通常是一个 Error 实例。

InteratorResult接口

所有迭代器的方法（next()、return()、throw()）都应返回实现InteratorResult的接口，接口定义如下：

interface IteratorResult {
  done?: boolean;
  value?: any;
}

即，当调用next()函数时，只可能返回两种情况：

  1. 迭代未结束时，返回：
  {
    "done": false,
    "value": "任何值"
  }
  1. 迭代结束时，返回：
  {
    "done": true,
    "value": undefined,
  }

可迭代迭代器

一个迭代器实现了[Symbol.iterator]()方法，并返回this，则此迭代器变为可迭代迭代器。

const myIterator = {
  next() {
    // ...
  },
  [Symbol.iterator]() {
    return this;
  }
}

大部分内置的语法或者API，都期望接受一个可迭代迭代器作为参数。如：Map()、Set()、Promise.all()、Promise.allSettled()、Promise.any()等，见下文。

迭代器

Note

迭代器维护着一个指向可迭代对象的引用，因此迭代器会阻止垃圾回收程序回收可迭代对象

检查是否可迭代

很多内置类型都实现了 Iterable 接口，比如：

  • 字符串
  • 数组
  • 映射
  • 集合
  • arguments对象
  • NodeList等DOM集合类型

检查是否存在默认迭代器属性（Symbol.iterator）可以暴露这个工厂函数：

let str = 'Hello world!';
let obj = {};

console.log(str[Symbol.iterator]); // ƒ [Symbol.iterator]() { [native code] }
console.log(obj[Symbol.iterator]); // undefined

// 调用这个工厂函数会生成一个迭代器
let iterator = str[Symbol.iterator]();
console.log(iterator); // StringIterator {}

支持迭代器的原生语言

实际写代码过程中，不需要显式调用这个工厂函数来生成迭代器。在一些原生语言中，会自动兼容接收可迭代对象的特性。接收可迭代对象的原生语言特性包括：

  • for...of循环
  • 数组解构
  • 扩展操作符
  • Array.from()
  • 创建Set、WeakSet
  • 创建Map、WeakMap
  • Promise.all()
  • Promise.race()
  • Promise.allSettled()
  • Promise.any()
  • Object.groupBy()
  • Map.groupBy()
  • yield*操作符，在生成器中使用

这些原生语言结构会自动调用提供的可迭代对象的这个工厂函数，从而创建一个迭代器:

let iterableObj = {
  data: ['Javascript', 'Java', 'Rust', 'Python'],
  // 实现可迭代的协议
  [Symbol.iterator]() {
    let index = 0;
    let data = this.data;

    // 实现 Iterator 接口，返回新的迭代器对象
    return {
      // 实现迭代器协议
      next() {
        if (index < data.length) {
          return { value: data[index++] };
        }
        return { done: true };
      }
    }
  }
}

/**
 * for...of 循环
*/
for (let item of iterableObj) {
  console.log(item);
}
// 'Javascript'
// 'Java'
// 'Rust'
// 'Python'

/**
 * 数组解构
*/
let [a, b, c, d] = iterableObj;
console.log(a, b, c, d); // Javascript Java Rust Python

/**
 * 扩展操作符
*/
console.log([...iterableObj]); // ['Javascript', 'Java', 'Rust', 'Python']

/**
 * Array.from()
*/
console.log(Array.from(iterableObj)); // ['Javascript', 'Java', 'Rust', 'Python']

/**
 * Set 构造函数
*/
let set = new Set(iterableObj);
console.log(set); // Set(4) {'Javascript', 'Java', 'Rust', 'Python'}

/**
 * Map 构造函数
*/
let pairs = Array.from(iterableObj).map((value, index) => [index, value]);
let map = new Map(pairs);
console.log(map); // Map(4) {0 => 'Javascript', 1 => 'Java', 2 => 'Rust', 3 => 'Python'}

让类成为迭代器

要让类变成迭代器，需符合可迭代协议:

class Range {
  constructor(from, to) {
    this.from = from;
    this.to   = to;
  }

  [Symbol.iterator]() {
    let next = Math.ceil(this.from);
    let last = this.to;

    return {
      next() {
        if (next <= last) {
          return { value: next++ };
        }
        return { done: true };
      }
    }
  }
}

// eg
for (let x of new Range(1, 10)) {
  console.log(x); // log 1~10
}
console.log(...new Range(-2, 2)); // -2,-1,0,1,2

一般情况下，都会利用闭包的机制，使得一个可迭代对象能够创建多个迭代器。

函数返回可迭代

除了让类可迭代，还可以让函数返回可迭代的迭代器。

示例：map函数

function map(iterable, fn) {
  let iterator = iterable[Symbol.iterator]();

  return {
    [Symbol.iterator]() { return this; },
    next() {
      let result = iterator.next();
      if (result.done) {
        return result;
      }
      return { value: fn(result.value) };
    },
  }
}

// usage
console.log(...map(new Range(1,4), x => x * x)) // 1,4,9,16

示例：filter函数

function filter(iterable, predicate) {
  let iterator = iterable[Symbol.iterator]();

  return {
    [Symbol.iterator]() { return this; },
    next() {
      for(;;) {
        let result = iterator.next();
        if (result.done || predicate(result.value)) {
          return result;
        }
      }
    }
  }
}

// usage
console.log(...filter(new Range(1, 10), x => x % 2 === 0)); // 2 4 6 8 10

提前终止迭代器

可选return()方法用于指定在迭代器提前关闭时执行的逻辑。即迭代器未完全迭代便提前退出，可能情况为：

  • for...of循环通过break、continue、return或throw提前退出；
  • 解构操作并未消费所有值；

Note

return()方法必须返回一个有效的 IteratorResult 对象，一般可以只返回{ done: true }，这个返回值只会用在生成器的上下文中。

如下代码，JS发现还有更多值可以迭代但不会消费这些值时，会自动调用return()方法。

let iterableObj = {
  data: ['Javascript', 'Java', 'Rust', 'Python'],
  [Symbol.iterator]() {
    let index = 0;
    let data = this.data;

    return {
      next() {
        if (index < data.length) {
          return { value: data[index++] };
        }
        return { done: true };
      },
      return() {
        console.log('iterableObj迭代提前终止');
        return { done: true };
      },
    }
  }
}

for (let item of iterableObj) {
  if (item === 'Java') {
    break;
  }
  console.log(item);
}
// 打印：'Javascript' 'iterableObj迭代提前终止'

try {
  for (let item of iterableObj) {
    if (item === 'Java') {
      throw 'err';
    }
    console.log(item);
  }
} catch(e) {}
// 打印：'Javascript' 'iterableObj迭代提前终止'

// 解构部分值
let [a, b] = iterableObj;
// 打印：'iterableObj迭代提前终止'

不可终止的迭代器

有些迭代器是不可提前终止的，比如数组的迭代器就是不可终止的，如果迭代器没有终止，则还可以从上次离开的地方继续迭代：

let a = [1, 2, 3, 4, 5];
let iter = a[Symbol.iterator]();
for (let i of iter) {
  console.log(i);
  if (i > 2) {
    break;
  }
}
// 1
// 2
// 3

for (let i of iter) {
  console.log(i);
}
// 4
// 5

一个不可终止的迭代器即使添加了return()方法，也并不会将其变为可终止。这种情况下，return()函数可以正常被执行，但迭代器不会被强制终止。

let a = [1, 2, 3, 4, 5];
let iter = a[Symbol.iterator]();

iter.return = function () {
  console.log('提前退出');
  return { done: true };
};

for (let i of iter) {
  console.log(i);
  if (i > 2) {
    break;
  }
}
// 1
// 2
// 3
// 提前退出

for (let i of iter) {
  console.log(i);
}
// 4
// 5

无限迭代器

可以创建一个永远不会主动终止的迭代器：

let idIterable = {
  [Symbol.iterator]() {
    let id = 0;

    return {
      next() {
        return { done: false, value: id++ }
      }
    }
  }
}

const idCreator = idIterable[Symbol.iterator]();
console.log(idCreator.next()); // {done: false, value: 0}
console.log(idCreator.next()); // {done: false, value: 1}
console.log(idCreator.next()); // {done: false, value: 2}

// 可以提前终止一个无限迭代器
for (let id of idIterable) {
  console.log(id);
  if (id === 10) {
    break;
  }
}
// 打印1~10

内置的JS迭代器

以下都是内置的Javascript迭代器，摘自Iterator:

  • 数组迭代器，返回自Array.prototype.values()、Array.prototype.keys()、Array.prototype.entries()、Array.prototype[Symbol.iterator]()、TypedArray.prototype.values()、TypedArray.prototype.keys()、TypedArray.prototype.entries()、TypedArray.prototype[Symbol.iterator]() 和 arguments[Symbol.iterator]()
  • 字符串迭代器，返回自String.prototype[Symbol.iterator]()
  • Map 迭代器，返回自Map.prototype.values()、Map.prototype.keys()、Map.prototype.entries() 和 Map.prototype[Symbol.iterator]()
  • Set 迭代器，返回自Set.prototype.values()、Set.prototype.keys()、Set.prototype.entries() 和 Set.prototype[Symbol.iterator]()
  • 正则表达式字符串迭代器，返回自RegExp.prototype[Symbol.matchAll]() 和 String.prototype.matchAll()
  • Generator 对象，返回自生成器函数
  • Segment迭代器，返回自 Intl.Segmenter.prototype.segment() 返回的 Segments 对象的 [Symbol.iterator]() 方法
  • 迭代器辅助方法，返回自迭代器辅助方法例如 Iterator.prototype.filter() 和 Iterator.prototype.map()

使用迭代器作为可迭代对象

所有内置迭代器都是可迭代的，因此可以在for...of循环中使用：

const arrIterator = [1, 2, 3].values();
for (const value of arrIterator) {
  console.log(value);
}
// 打印：1, 2, 3

迭代器辅助函数

迭代器辅助函数（iterator helpers）用于提升开发者的工作效率。Iterator 类本身提供了一些使用迭代器的辅助方法。 比如以下代码：

const nameToDeposit = new Map([
  ["Anne", 1000],
  ["Bert", 1500],
  ["Carl", 2000],
]);

const totalDeposit = [...nameToDeposit.values()].reduce((a, b) => a + b);

可以直接简化为：

const totalDeposit = nameToDeposit.values().reduce((a, b) => a + b);

减少中间数组的使用，且只迭代迭代器一次。

常见的迭代器辅助方法有：

迭代器辅助方法 说明 返回值 类型
Iterator.prototype.drop() 跳过前n个值 Iterator 惰性
Iterator.prototype.every() 全部满足条件判断 Boolean 急切
Iterator.prototype.filter() 只保留满足条件的值 Iterator 惰性
Iterator.prototype.find() 查找首个匹配值，无匹配则为undefined Value 急切
Iterator.prototype.flatMap() 映射后展平一层 Iterator 惰性
Iterator.prototype.forEach() 逐一执行副作用 undefined 急切
Iterator.prototype.map() 对每个值应用变换 Iterator 惰性
Iterator.prototype.reduce() 规约为单个值 Iterator 急切
Iterator.prototype.some() 存在性判断 Boolean 急切
Iterator.prototype.take() 只取前n个值 Iterator 惰性
Iterator.prototype.toArray() 转换为数组 Array 急切
Iterator.prototype[Symbol.iterator]() 返回迭代器对象本身。这使迭代器对象也是可迭代的 this --

Note

这些方法是迭代器辅助方法，而不是可迭代对象辅助方法，因为可迭代对象的唯一要求就是具有 [Symbol.iterator]() 方法，因此它们没有共享的原型来安装这些方法。

还有一些静态方法：

静态方法 说明 返回值
Iterator.from() 将迭代器或者可迭代对象转为“恰当”的迭代器 Iterator

惰性求值

迭代器辅助函数默认是惰性的。

传统的数组方法会立即处理所有内容：

// 急切求值 - 立即处理所有元素
const eager = [1, 2, 3, 4, 5]
  .map(x => {
    console.log(`Mapping ${x}`)
    return x * 2
  })
  .filter(x => x > 5)

// 输出：Mapping 1, 2, 3, 4, 5
// 结果：[6, 8, 10]

而迭代器辅助函数只在消费时处理值：

// 惰性求值 - 只处理需要的内容
const lazy = [1, 2, 3, 4, 5]
  .values()
  .map(x => {
    console.log(`Mapping ${x}`)
    return x * 2
  })
  .filter(x => x > 5)
  .take(2)

// 还没有任何输出！

const result = [...lazy]
// 输出：Mapping 1, 2, 3
// 结果：[6, 8]

注意惰性版本在找到两个匹配值后就停止了，从不处理元素 4 和 5。这种效率在处理大数据集时变得至关重要。

性能与内存

迭代器辅助函数在以下场景下表现出色：

  • 处理大于可用内存的数据
  • 只需要结果的子集
  • 多个链式转换
  • 处理流或者实时数据

在以下场景下，则不太适用：

  • 需要随机访问元素
  • 数据集很小且已在内存中
  • 需要多次遍历（迭代器是单次使用的，除非创建一个迭代器工厂函数）

可以使用下面的代码进行测试（100万条数据，取前5条）：

// 测试代码
const logs = Array.from({ length: 1_000_000 }, (_, i) => ({
  level: i % 100 === 0 ? 'error' : 'info',
  message: `Log entry #${i}`,
}));

// 测试1: 传统 Array 方法
console.time('Array methods');
const result1 = logs
  .filter(log => log.level === 'error')
  .slice(0, 5)
  .map(log => `[${log.level}] ${log.message}`);
console.timeEnd('Array methods');

// 测试2: Iterator Helpers
console.time('Iterator Helpers');
const result2 = logs.values()
  .filter(log => log.level === 'error')
  .take(5)
  .map(log => `[${log.level}] ${log.message}`)
  .toArray();
console.timeEnd('Iterator Helpers');

// 测试3: 手写 for 循环
console.time('Manual loop');
const result3 = [];
for (const log of logs) {
  if (log.level === 'error') {
    result3.push(`[${log.level}] ${log.message}`);
    if (result3.length === 5) break;
  }
}
console.timeEnd('Manual loop');

可以参考以下数据（摘自网络，环境为 Nodejs 22.0/V8 12.4）进行对比：

方法 执行时间 内存分配 元素遍历数
Array methods ~45ms ~800KB（中间数组） 1,000,000
Iterator Helpers ~0.8ms ~0KB（无中间数组） ~401
Manual loop ~0.6ms ~0KB ~401

结论：迭代器辅助函数的性能接近手写循环（仅慢30%左右），但相比传统Array方法，快50倍以上，且没有中间内存分配。

示例：API分页

高效处理分页API（这里使用了Async Iterator Helpers）：

async function* fetchAllUsers(apiUrl) {
  let page = 1
  while (true) {
    const response = await fetch(`${apiUrl}?page=${page}`)
    const { data, hasMore } = await response.json()
    
    for (const user of data) yield user
    
    if (!hasMore) break
    page++
  }
}

// 处理用户而不加载所有页面
const premiumUsers = await fetchAllUsers('/api/users')
  .filter(user => user.subscription === 'premium')
  .map(user => ({ id: user.id, email: user.email }))
  .take(50)
  .toArray()
示例：文件流处理

逐行处理大文件，不用将整个文件加载到内存中：

async function* readLines(file) {
  const reader = file.stream().getReader()
  const decoder = new TextDecoder()
  let buffer = ''
  
  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    
    buffer += decoder.decode(value, { stream: true })
    const lines = buffer.split('\n')
    buffer = lines.pop()
    
    for (const line of lines) yield line
  }
  if (buffer) yield buffer
}

// 处理 CSV 而不加载整个文件
const validRecords = await readLines(csvFile)
  .drop(1)  // 跳过标题行
  .map(line => line.split(','))
  .filter(cols => cols[2] === 'active')
  .take(100)
  .toArray()
示例：数据管道处理CSV解析与转换
// 解析 CSV 行的生成器
function* parseCSV(text) {
  for (const line of text.split('\n')) {
    if (line.trim()) {
      yield line.split(',').map(cell => cell.trim());
    }
  }
}

const csvData = `
name,age,city
Alice,30,NYC
Bob,25,LA
Carol,35,SF
Dave,28,NYC
`;

// 提取所有 NYC 居民的姓名和年龄
const nycResidents = Iterator.from(parseCSV(csvData))
  .drop(1)  // 跳过表头
  .filter(([name, age, city]) => city === 'NYC')
  .map(([name, age]) => ({ name, age: parseInt(age) }))
  .toArray();
// → [{ name: 'Alice', age: 30 }, { name: 'Dave', age: 28 }]
示例：处理无限序列

生成和处理无限数据流：

function* fibonacci() {
  let [a, b] = [0, 1]
  while (true) {
    yield a
    ;[a, b] = [b, a + b]
  }
}

// 找到第一个大于 1000 的斐波那契数
const firstLarge = fibonacci()
  .find(n => n > 1000)  // 1597

// 获取前 10 个偶数斐波那契数
const evenFibs = fibonacci()
  .filter(n => n % 2 === 0)
  .take(10)
  .toArray()

常见陷阱

常见陷阱：迭代器是单次可用的
const iter = [1, 2, 3].values().map(x => x * 2)
console.log([...iter])  // [2, 4, 6]
console.log([...iter])  // [] - Already consumed!

// Solution: Create a new iterator
const makeIter = () => [1, 2, 3].values().map(x => x * 2)
常见陷阱：迭代器和数组方法混用
// Won't work - filter returns iterator, not array
const result = [1, 2, 3]
  .values()
  .filter(x => x > 1)
  .includes(2)  // Error!

// Solution: Convert back to array first
const result = [1, 2, 3]
  .values()
  .filter(x => x > 1)
  .toArray()
  .includes(2)  // true

迭代器辅助函数的原理

迭代器辅助函数如何做到惰性求值？为什么链式调用不会立即执行？

答案在于Generator 机制的复用。ECMAScript 规范将每个 iterator helper 都实现为一个 Generator ，利用 Generator 的暂停/恢复能力实现惰性执行。

惰性求值具备如下特点：

  • 声明不执行：调用map()/filter()/take()等，仅返回新的迭代器，不遍历、不计算、不生成中间数组；
  • 消费才触发：仅在调用toArray()/for...of/next()等操作时，才从源头开始逐元素处理；
  • 按需逐元素：每次next()只处理一个元素，管道逐级传递，无需全量加载；
  • 提前终止：调用take()收集够n个有效元素后立即停止，后续元素不处理；
  • 拉取模型：数据由末端按需从上游逐个拉取；

与Generator的关系

iterator helper 约等于受限的Generator：

// 手写 Generator 实现 map
function* manualMap(iterator, fn) {
  for (const value of iterator) {
    yield fn(value);
  }
}

// 等价于
iterator.map(fn);

但iterator helper有如下限制：

  1. 不支持.throw()，helper不暴露throw方法，不能向管线中注入异常；
  2. 不转发.next()的参数，helper.next(someValue)中的someValue参数会被忽略；
  3. .return()会关闭底层迭代器，调用helper.return()会沿着链路关闭所有上游迭代器。

即，iterator helper虽然底层是 Generator 机制，但对外只暴露纯粹的Iterator接口。

执行流程解析

以以下代码为例：

// 1. 声明阶段：惰性包装，不执行任何逻辑
const iterator = [1,2,3,4,5]
  .values() // 原始迭代器
  .filter(x => x > 2) // 包装为 filterIter
  .map(x => x * 2) // 包装为 mapIter
  .take(2); // 包装为 takeIter

// 2. 消费阶段：才真正开始执行（唯一触发点）
console.log(Array.from(iterator)); // [6, 8]

执行的流程为：

  • 每个方法返回新迭代器，保存回调与上游迭代器引用
  • 消费执行阶段（Array.from() 触发），下游拉取 → 上游响应（take → map → filter → 源头）
    • 请求第一个结果: 取 1 → 过滤失败 → 取 2 → 过滤失败 → 取 3 → 过滤通过 → 乘 2 → take 计数 1
    • 请求第二个结果: 取 4 → 过滤通过 → 乘 2 → take 计数 2
    • 终止: 直接停止，5 不会被读取和处理

手写简化版

理解了规范算法后，用js实现一个简化版，加深对底层机制的理解：

class LazyIterator {
  #source;

  constructor(source) {
    // source 是一个 { next() } 对象
    this.#source = source;
  }

  next() {
    return this.#source.next();
  }

  [Symbol.iterator]() {
    return this;
  }

  return() {
    return this.#source.return?.() ?? { value: undefined, done: true };
  }

  // -- 惰性方法 --
  map(mapper) {
    const source = this;
    let counter = 0;

    return new LazyIterator({
      next() {
        const { value, done } = source.next();
        if (done) {
          return { value: undefined, done: true };
        }
        return { value: mapper(value, counter++), done: false }
      },
      return() {
        return source.return?.() ?? { value: undefined, done: true };
      }
    });
  }

  filter(predicate) {
    const source = this;
    let counter = 0;

    return new LazyIterator({
      next() {
        while(true) {
          const { value, done } = source.next();
          if (done) {
            return { value: undefined, done: true };
          }
          if (predicate(value, counter++)) {
            return { value, done: false }
          }
          // 不满足条件，则继续拉取下一个，filter内部的循环
        }
      },
      return() {
        return source.return?.() ?? { value: undefined, done: true };
      }
    });
  }

  take(limit) {
    if (limit <= 0) {
      return new LazyIterator({
        next() { return { value: undefined, done: true }; },
        return() { return { value: undefined, done: true }; }
      });
    }

    const source = this;
    let remaining = limit;
    let closed = false;

    return new LazyIterator({
      next() {
        if (!closed && remaining <= 0) {
          // 提前终止
          source.return?.();
          closed = true; // 防止重复关闭
        }
        if (remaining <= 0) {
          return { value: undefined, done: true };
        }
        remaining--;
        return source.next();
      },
      return() {
        if (!closed) {
          closed = true;
          return source.return?.() ?? { value: undefined, done: true };
        }
        return { value: undefined, done: true };
      },
    });
  }

  // -- 急切方法 --
  toArray() {
    const result = [];
    while(true) {
      const { value, done } = this.next();
      if (done) return result;
      result.push(value);
    }
  }

  // ...其他方法，实现类似，此处省略

  // 静态工厂方法
  static from(iteratorOrIterable) {
    if (iteratorOrIterable == null) {
      throw new TypeError('Cannot convert null or undefined to iterator');
    }
    // 先检查是否已经是迭代器（有 .next() 方法）
    if (typeof iteratorOrIterable.next === 'function') {
      return new LazyIterator(iteratorOrIterable);
    }
    // 再检查是否是可迭代对象（有 Symbol.iterator）
    if (typeof iteratorOrIterable[Symbol.iterator] === 'function') {
      return new LazyIterator(iteratorOrIterable[Symbol.iterator]());
    }
    throw new TypeError('Argument must be an iterator or iterable object');
  }
}

验证：

function naturals() {
  return {
    [Symbol.iterator]() {
      let n = 1;
      return {
        next() {
          return { value: n++, done: false }
        }
      }
    }
  }
}

const result = LazyIterator.from(naturals())
  .filter(n => n % 2 === 0)
  .map(n => n * 10)
  .take(5)
  .toArray();

console.log(result);
// → [20, 40, 60, 80, 100]
