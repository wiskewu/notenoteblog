修改所有提交的commit信息

2026/3/3 01:27:0110 分钟阅读
Git

背景

在公司工作时，通常会有两套 git 账户：

  • 工作账户：一个用于公司内网开发项目提交的账户
  • 私人账户：一个用于个人在 Github 网站上的账户

在开发过程中，经常会由于忘记设置账户名导致在私人项目中使用工作账户进行 commit 提交的操作。当本地开发一段时间后，想要将个人项目提交至 Github 时，才发现 commit 信息仍然是工作账户。

此时，就需要一种方法，能够将这些历史提交信息中的账户改写为私人账户信息。

方案

在文章如何修改Git提交历史中的author，email和name等信息中一共提及两种方式：

  • 方案一：git rebase
  • 方案二：git filter-branch

由于方案一变基比较繁琐，本文只对方案二git filter-branch进行实践操作和记录。

以下假设账户分别为：

  • 工作账户：user.name=workman user.email=workman@example.com
  • 私人账户：user.name=freeman user.email=freeman@example.com

filter-branch

在当前项目根目录创建git-commit-rewrite.sh脚本（使用时需替换实际用户信息）：

# !/bin/sh

git filter-branch --env-filter '

OLD_EMAIL="workman@example.com"
CORRECT_NAME="freeman"
CORRECT_EMAIL="freeman@example.com"

if [ "$GIT_COMMITTER_EMAIL" = "$OLD_EMAIL" ]

then

    export GIT_COMMITTER_NAME="$CORRECT_NAME"
    export GIT_COMMITTER_EMAIL="$CORRECT_EMAIL"

fi

if [ "$GIT_AUTHOR_EMAIL" = "$OLD_EMAIL" ]

then

    export GIT_AUTHOR_NAME="$CORRECT_NAME"
    export GIT_AUTHOR_EMAIL="$CORRECT_EMAIL"

fi

' --tag-name-filter cat -- --branches --tags

然后在终端（当前项目根目录中打开）执行：

bash ./git-commit-rewrite.sh

至此，所有commit的信息就修改完毕了。

git-filter-repo

使用filter-branch时，在终端会提示以下信息：

Warning

WARNING: git-filter-branch has a glut of gotchas generating mangled history rewrites. Hit Ctrl-C before proceeding to abort, then use an alternative filtering tool such as 'git filter-repo' (https://github.com/newren/git-filter-repo/) instead. See the filter-branch manual page for more details; to squelch this warning, set FILTER_BRANCH_SQUELCH_WARNING=1. Proceeding with filter-branch...

git官方推荐使用类似工具git-filter-repo去实现这个功能，于是我也对这个工具进行了尝试。

安装

前置条件，需安装git和python至电脑：

  • git >= 2.36.0
  • python3 >= 3.6

然后根据官方文档INSTALL所示，下载文件（下载时需注意保留无后缀形式）。

我下载的文件保存至"C:\Users\DELL\Downloads\git-filter-repo"位置。

使用

在项目中打开终端，运行以下命令（使用时需替换实际用户信息）：

python C:\\Users\\DELL\\Downloads\\git-filter-repo \
  --name-callback 'return b"freeman" if name == b"workman" else name' \
  --email-callback 'return b"freeman@example.com" if email == b"workman@example.com" else email' \
  --force

Warning

此操作会提示以下信息： NOTICE: Removing 'origin' remote; see 'Why is my origin removed?' in the manual if you want to push back there.

表明远端仓库源被移除，具体移除原因参见使用手册。 可使用以下命令重新添加远端源: git remote add origin $ORIGINAL_CLONE_URL，"$ORIGINAL_CLONE_URL"表示远端仓库地址。

注意事项

如需将修改信息同步至远端仓库，可使用git push -f命令进行强制同步。该操作会覆盖远端分支的提交历史，请自行确认操作风险。 提交之后，若需保证当前项目后续提交信息一致，仍需设置项目级的用户信息，见下文。

为每个项目单独设置用户名和邮箱

在项目根目录位置打开终端，执行以下命令：

# 替换freeman为实际用户名
git config --local user.name freeman
# 替换freeman@example.com为实际邮箱地址
git config --local user.name freeman@example.com

# 查询
git config --local -l
