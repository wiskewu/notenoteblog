nodejs创建单个可执行应用程序

2026/5/12 20:48:4215 分钟阅读
WindowsNodejs

Node.js SEA 是实验性能力（稳定性 1.1），原生支持创建单个可执行应用程序（Single Executable Applications，简称SEA），用于把 Node.js 应用打包成不依赖系统安装 Node 的独立可执行文件，核心是将 脚本/资源 注入 node 二进制，启动时自动运行内嵌代码。这个特性使得开发者可以将 Nodejs 应用分发至没有安装 Nodejs 的系统上。

工作原理

Node.js 允许将一个由 Node.js 准备的blob（数据块）注入到 Node 二进制文件中，这个"blob"可以包含被打包的脚本。 应用启动时，会检查是否有所注入的内容。如果找到了这个"blob"，它就会执行其中的脚本；如果没有，则 Node.js 按常规方式运行。

创建流程

可以将流程分为以下几步：

Note

以下流程仅在Windows环境下进行测试，MacOS可能存在部分差异。

步骤一：创建js文件

例如，创建一个打印“Hello”的 Nodejs 脚本：

console.log(`Hello, ${process.argv[2]}!`);

步骤二：创建配置文件

创建一个dist/sea-config.json文件，指定主脚本文件名及其输出的V8快照blob：

{
  "main": "src/index.js",
  "output": "dist/sea-prep.blob",
  "disableExperimentalSEAWarning": true
}

步骤三：生成blob

使用 Nodejs 的实验性标记 --experimental-sea-config 和上述配置将主脚本及其依赖打包为单个V8快照blob：

node --experimental-sea-config dist/sea-config.json

执行后，生成一个blob文件位于dist/sea-prep.blob。

步骤四：复制nodejs二进制文件

创建一个可执行的 Node 副本，并根据需要重命名（例如my-app.exe）：

const fs = require('fs');
// 复制当前的node.exe，重命名为my-app.exe
fs.copyFileSync(process.execPath, 'dist/my-app.exe');

步骤五：注入blob

Note

在MacOS上，执行此步骤之前，可能还需要移除复制的二进制文件的签名，才可以进行blob注入。

使用postject将上述blob快照注入到 Node 副本：

npx postject dist/my-app.exe NODE_SEA_BLOB dist/sea-prep.blob --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6e072b8b5df1996b2 --overwrite

命令参数解释如下：

  • postject：一款注入工具
  • dist/my-app.exe：Nodejs副本，可独立运行，无需Nodejs运行时
  • NODE_SEA_BLOB dist/sea-prep.blob：步骤三中创建的blob
  • --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6e072b8b5df1996b2：Nodejs 项目用于检测是否注入了文件
  • --overwrite：允许覆盖已有的blob

简化流程（V25.5.0）

上述流程是旧版本的操作，自 Nodejs V25.5.0 版本开始，可以使用下述方式快速创建 SEA：

  1. 写入口 JS
  2. 创建 sea-config.json
  3. 执行node --build-sea sea-config.json
  4. 按需签名（macOS/Windows）
  5. 直接运行生成的 exe

具体见官方文档

sea-config配置

字段 作用 默认 / 可选
main 入口脚本路径 必填
mainFormat 模块类型 commonjs / ES module
output 输出可执行文件路径 必填
executable 用于注入的 node 二进制 当前 node
disableExperimentalSEAWarning 关闭实验警告 false
useSnapshot 启用启动快照 false
useCodeCache 启用代码缓存加速启动 false
execArgv 内置 Node 启动参数 -
execArgvExtension 参数扩展方式 env / none / cli
assets 内嵌资源（图片 / 文本等） 可选

关键能力

资源内嵌

配置中可以指定资源路径，打包进 blob，然后通过node:sea模块读取：

  • 支持的方法有：getAsset()、getAssetAsBlob()、getRawAsset()、getAssetKeys()

启动优化

通过配置下述字段，可以加速启动：

  • useCodeCache: 预编译代码缓存，提速启动；此字段设置后，在脚本中不支持import()语法
  • useSnapshot: 构建时生成快照，运行时反序列化；需配置v8.startupSnapshotAPI使用

执行参数控制

  • execArgv: 固化 Node 参数（如--max-old-space-size）
  • execArgvExtension: 控制是否允许 NODE_OPTIONS 或 --node-options 传参

原生模块加载

  • 把 .node 作为 assets 内嵌
  • 运行时写出到临时文件，用 process.dlopen() 加载

重要限制与注意事项

在使用 SEA 时，需要注意以下几点：

  • 跨平台构建时，必须关闭 useSnapshot和 useCodeCache
  • module 模式不可与 useSnapshot 共用
  • 内嵌脚本默认只能加载内置模块，需 createRequire 才能读取文件系统
  • __filename和 __dirname 指向可执行文件本身，非原始 JS 路径
  • linux arm64容器环境构建可能导致原生模块加载失败

node:sea 核心API

  • sea.isSea(): 是否运行在SEA环境
  • getAsset(key, encoding?): 读取资源（字符串/ArrayBuffer）
  • getRawAsset(key): 无拷贝取原始资源
  • getAssetAsBlob(key, options?): 以 Blob 形式取资源
  • getAssetKeys(): 获取所有资源键名

结合web的案例

SEA可以启动后端服务（例如express、或者nodejs原生 http），在应用打开时，自动在浏览器打开locahost:3000/页面。

此时对于web静态资源的托管，有两种方案:

  1. 通过SEA的assets配置，将静态资源全部打包进exe中
  2. 额外托管一个public（放置web静态资源）目录，通过后端读取磁盘文件

方案一优缺点

把 html/js/css/图片 全部写进 sea-config.json 的 assets 里，打包进 exe 内部，优点：

  • 最终只有 一个 exe 文件，无额外文件夹
  • 部署、分发简单
  • 不会因为文件丢失 / 路径错误导致页面打不开
  • 后端直接用 node:sea API 读取，速度极快

缺点是：

  • 修改静态资源必须重新打包
  • 适合前端资源不频繁变动的项目

方案二优缺点

后端服务去读 exe 程序同级 public 文件夹，优点是：

  • 改静态资源不用重新打包

缺点是：

  • 必须附带 public/ 文件夹，不是单文件
  • 路径容易出错
  • 不符合 SEA 单执行文件的初衷
  • 分发麻烦（用户必须拷走整个文件夹）

最佳实践

最推荐直接用 SEA 的 assets 能力托管静态文件。

项目结构示例如下：

your-sea-app/
├── sea-config.json
├── server.js        # 后端入口
└── public/          # 静态资源
    ├── index.html
    ├── js/app.js
    └── css/style.css

sea-config.json 配置如下（需包含所有前端资源）：

{
  "main": "server.js",
  "output": "app.exe",
  "assets": {
    "/": "public/index.html",
    "/js/app.js": "public/js/app.js",
    "/css/style.css": "public/css/style.css"
  }
}

express后端服务示例代码：

import express from 'express';
import { getAsset } from 'node:sea';

const app = express();

// 托管静态资源
app.get('*', (req, res) => {
  try {
    // 从 SEA 内嵌资源里读取
    const content = getAsset(req.path, 'utf8');
    
    // 根据后缀自动设置 Content-Type
    if (req.path.endsWith('.html')) res.setHeader('Content-Type', 'text/html');
    if (req.path.endsWith('.js')) res.setHeader('Content-Type', 'application/javascript');
    if (req.path.endsWith('.css')) res.setHeader('Content-Type', 'text/css');

    res.send(content);
  } catch (e) {
    res.status(404).send('Not Found');
  }
});

app.listen(3000, () => {
  console.log('SEA 服务启动：http://localhost:3000');
});
