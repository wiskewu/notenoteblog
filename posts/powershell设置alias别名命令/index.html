PowerShell设置Alias别名命令

2026/4/6 01:40:17 · 更新于2026/4/10 17:31:385 分钟阅读
Windows

在 PowerShell 中设置快捷别名（Alias）主要有临时生效和永久生效两种方式，核心命令是 Set-Alias（或 New-Alias），永久生效需写入 PowerShell 配置文件（Profile）。

此文只记录永久设置的方式。

写入永久别名规则

以将$env:OPENCODE_CONFIG="E:/githubrefs/opencodecfg/opencode.jsonc"; $env:OPENCODE_CONFIG_DIR="E:/githubrefs/opencodecfg/prod"; opencode 设置为别名 myopencode 为例。

找到或创建配置文件

# 查看当前用户配置文件路径
$PROFILE
# 输出示例：C:\Users\DELL\Documents\PowerShell\Microsoft.PowerShell_profile.ps1

# 若文件不存在，创建它
New-Item -Type File -Path $PROFILE -Force

编辑配置文件

用记事本打开并添加别名：

# 打开配置文件
notepad $PROFILE

写入规则

在配置文件写入以下规则：

function myopencode {
    $env:OPENCODE_CONFIG = "E:/githubrefs/opencodecfg/opencode.jsonc"
    $env:OPENCODE_CONFIG_DIR = "E:/githubrefs/opencodecfg/prod"
    opencode
}

保存并关闭。

生效配置

# 立即加载配置（无需重启 PowerShell）
. $PROFILE

# 或重启 PowerShell 自动生效

在PowerShell 中输入 myopencode 即可启动带有参数的 opencode。

引申了解

临时别名

临时别名仅在当前打开的 PowerShell 窗口生效，关闭后消失。 基础命令：Set-Alias / sal

# 语法
Set-Alias -Name <别名> -Value <原命令/程序>
# 简写（推荐）
sal <别名> <原命令>

常用示例：

# 1. 系统命令别名（如 ls → Get-ChildItem）
sal ls Get-ChildItem
sal ll "Get-ChildItem -Force"  # 带参数需用引号

# 2. 外部程序别名（如 np → 记事本）
sal np notepad
sal code "C:\Program Files\Microsoft VS Code\Code.exe"

# 3. 查看所有别名
Get-Alias
# 查看指定别名
Get-Alias ls

导出/导入别名:

# 导出所有别名到文件
Export-Alias -Path "D:\alias_backup.txt"

# 导入别名（覆盖现有）
Import-Alias -Path "D:\alias_backup.txt" -Force
