yarn设置注册表源

2026/4/21 15:51:066 分钟阅读
yarn

背景

在使用yarn config set registry "http://192.168.25.57:8081/repository/npm-group/"，出现以下错误提示：

核心原因

排查后，发现是由于不同版本API变化导致的。当前使用的是 Yarn 2+（Berry，包括 v2/v3/v4），而 registry 配置项和 yarn config set registry 命令，是 Yarn 1.x（Classic）的用法，在新版本中已被废弃。

查看当前版本：

版本差异对比

特性 Yarn 1.x (Classic) Yarn 2+ (Berry)
配置文件 .yarnrc .yarnrc.yml
注册表配置项 registry npmRegistryServer
设置命令 yarn config set registry <url> 不支持直接用 registry，需修改配置文件或用新命令

Note

题外话，Yarn大版本更新，直接不考虑向后兼容了，不知道出于何种考虑，感觉有些粗暴。。。

解决办法

Yarn@2+(Berry)

方法一：直接修改项目配置文件

在项目根目录编辑或创建 .yarnrc.yml文件，添加以下内容：

npmRegistryServer: "http://192.168.25.57:8081/repository/npm-group/"
# 如需设置全局源，可编辑 ~/.yarnrc.yml

方法二：新版本命令

Yarn 2+ 中，注册表配置项名称改为 npmRegistryServer，可直接用 config set 设置：

yarn config set npmRegistryServer "http://192.168.25.57:8081/repository/npm-group/"

YN0001错误处理

设置完源之后，执行yarn安装依赖，出现以下错误信息：

这个 YN0001 错误是 Yarn 2+（Berry 系列，当前用的是 Yarn 3.2.4）的安全机制导致的：Yarn 默认会拒绝所有 HTTP（非 HTTPS）协议的请求，以防止中间人攻击。当前私有仓库地址 http://192.168.25.57:8081/... 使用了 HTTP，需要手动将该地址加入白名单。

添加白名单

修改项目根目录的 .yarnrc.yml 文件，在文件中添加 unsafeHttpWhitelist 配置，把私有仓库主机地址加入白名单：

# 你的私有仓库地址
npmRegistryServer: "http://192.168.25.57:8081/repository/npm-group/"

# 允许 HTTP 请求的白名单
unsafeHttpWhitelist:
  - "192.168.25.57"  # 直接写IP，也可以用通配符 "*.168.25.57"
  # 如需允许其他 HTTP 仓库，可继续添加
  # - "registry.npmmirror.com"

保存后，重新执行yarn即可。

Yarn@1+(Classic)

如果更习惯旧版用法，可以考虑降级：

# 卸载当前 Yarn（如果是通过 npm 安装的）
npm uninstall -g yarn
# 安装 Yarn 1.x 最新版
npm install -g yarn@1
# 验证版本
yarn --version # 输出 1.x.x 即为成功
# 再执行原命令即可生效
yarn config set registry http://192.168.25.57:8081/repository/npm-group/
