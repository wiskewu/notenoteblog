配置Git Bash作为Windows Terminal选项

2026/1/8 13:14:544 分钟阅读
Windows

Windows Terminal 默认识别到的终端可能不包含Git Bash，将Git Bash集成到终端中，能够极大方便终端管理和切换。

准备工作

首先确保电脑已经安装了Git软件。使用支持bash的终端查看：

where bash

#输出
# C:\Windows\System32\bash.exe
# C:\Users\DELL\AppData\Local\Microsoft\WindowsApps\bash.exe
# D:\Programs\Git\usr\bin\bash.exe

可以知道，D:\Programs\Git\usr\bin\bash.exe是Git Bash所在的目录。

配置

按照以下步骤操作：

  1. 打开Windows Terminal软件；
  2. 在标签页右侧下拉选项中，打开"设置"，或使用ctrl+,快捷键打开设置；
  3. 在左侧配置文件中，点击"添加新配置文件"-"新建空配置文件"；
  4. 名称填写"GitBash"；
  5. 命令行填写上述输出的"D:\Programs\Git\usr\bin\bash.exe --login"；
  6. 点击右下角的"保存"即可。

完整配置如下：

或使用json格式配置：

{
    "commandline": "D:\\Programs\\Git\\usr\\bin\\bash.exe --login",
    "guid": "{f1d2a988-6f53-41e2-b1f9-44c6d75135cd}",
    "hidden": false,
    "name": "GitBash"
}

Tip

命令行如果只配置"D:\Programs\Git\usr\bin\bash.exe"可能会导致bash shell能启动，但缺少必要的PATH环境变量，导致无法找到ls、ll、dir等命令。此时便可以添加--login参数以使用完整的bash功能。

使用

此时在终端的标签页下拉选项中就可以看到刚才新的GitBash选项，启动之后可以执行bash命令，如ll。
