动态加载远程umd格式的js代码

2026/3/3 01:18:4415 分钟阅读
Javascript

通常在开发编辑器/数据大屏/低代码等业务时，经常会面临一个问题：图元组件过多（第三方组件库代码作为独立项目单独打包），或者允许用户上传编写自定义代码逻辑时，如何在当前系统中加载第三方js代码？

一个普遍的做法是，将第三方代码打包成umd格式的代码，然后在当前系统中通过url链接形式加载js代码，并解析执行。

使用示例

UMDLoader.ts负责加载远程 UMD 组件（此处以React组件为例），并提供必要的依赖注入。

基础用法

假设模块打包后放置于服务器地址"/static/remote-demo/remote-demo-lib.umd.js"：

import { loadUMDModuleFromUrl } from 'path/to/UMDLoader'

// 从 URL 加载远程组件
const remoteModule = await loadUMDModuleFromUrl('/static/remote-demo/remote-demo-lib.umd.js')

if (remoteModule) {
  // 使用组件，假设模块导出组件和属性
  const Component = remoteModule.component
  const props = remoteModule.defaultProps

  // 渲染组件
  ReactDOM.createRoot(container).render(
    <Component {...props} />
  )
}

从文本加载

import { loadUMDModuleFromText } from 'path/to/UMDLoader'

const umdCode = `
  (function(factory) {
    // ... UMD 代码
  })(function(require, exports, module) {
    // ... 模块代码
  })
`

const module = await loadUMDModuleFromText(umdCode)

完整代码

UMDLoader.ts的完整实现如下：

/**
 * UMD 模块加载器
 */

import React from 'react'
import ReactDOM from 'react-dom'
import ReactDOMClient from 'react-dom/client'
import JsxRuntime from 'react/jsx-runtime'
import * as antd from 'antd'

/**
 * 公共模块
 * 提供给远程 UMD 组件使用的依赖
 * 注意此处会增大项目打包体积，因为是全量引入，之后尽量走全局CDN设置window.xxx
 */
export const COMMON_MODULE: Record<string, any | (() => any)> = {
  react: React,
  'react-dom': ReactDOM,
  'react-dom/client': ReactDOMClient,
  'react/jsx-runtime': JsxRuntime,
  antd: antd,
}

/**
 * 根据模块id获取公共模块
 * @param id 模块ID
 * @returns 模块对象
 */
export function getCommonModule(id: string): any {
  return COMMON_MODULE[id] || (window as any)[id]
}

/**
 * 从文本中加载 UMD 模块
 */
export async function loadUMDModuleFromText<T = any>(text: string): Promise<T | undefined> {
  return new Promise((resolve) => {
    try {
      if (!text) return undefined

      const module = { exports: {} }
      const require = (id: string) => getCommonModule(id)
      const fn = new Function('module', 'exports', 'require', text)

      try {
        fn(module, module.exports, require)
        const keys = Object.keys(module.exports)
        return resolve(keys.length ? (module.exports as T) : undefined)
      } catch (e) {
        console.warn('[loadUMDModuleFromText] 执行失败:', e)
        return resolve(undefined)
      }
    } catch (e) {
      console.warn('[loadUMDModuleFromText] 加载失败:', e)
      resolve(undefined)
    }
  })
}

/**
 * 从远程 URL 加载 UMD 模块
 */
export async function loadUMDModuleFromUrl<T = any>(url: string): Promise<T | undefined> {
  if (!url) return undefined

  try {
    const response = await fetch(url)
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`)
    }

    const text = await response.text()
    return await loadUMDModuleFromText<T>(text)
  } catch (e) {
    console.warn(`[loadUMDModuleFromUrl] 加载失败 (${url}):`, e)
    return undefined
  }
}

实现原理

一：依赖注入

需要实现一个require语法用于加载第三方公共依赖：

const require = (id: string) => getCommonModule(id)

getCommonModule 函数会：

  1. 首先在 COMMON_MODULE 中查找
  2. 如果找不到，在 window 全局对象中查找

其函数实现为：

export function getCommonModule(id: string): any {
  return COMMON_MODULE[id] || (window as any)[id]
}

二：模块执行

使用 new Function 创建沙箱环境执行 UMD 代码：

const module = { exports: {} }
const fn = new Function('module', 'exports', 'require', text)
fn(module, module.exports, require)

三：导出模块

返回模块 module.exports 对象：

return resolve(module.exports)

远程模块代码

格式要求

远程模块必须是标准的 UMD 格式：

(function(root, factory) {
  if (typeof exports === 'object' && typeof module === 'object') {
    // CommonJS
    module.exports = factory(require)
  } else if (typeof define === 'function' && define.amd) {
    // AMD
    define(['react'], factory)
  } else {
    // Browser globals
    root.RemoteDemo = factory(root.React)
  }
}(this, function(require) {
  // 模块代码，具体返回值类型由主项目约定
  return {
    type: 'remoteDemo',
    component: RemoteDemoComponent,
    defaultProps: { ... },
  }
}))

打包配置

在 Vite 打包时，配置打包格式为umd，并合理配置外部依赖声明（非必须，根据业务需求合理提取公共依赖）：

// vite.config.ts
export default defineConfig(() => {
  return {
    build: {
      lib: {
        entry: resolve(__dirname, 'src/umd-entry.ts'),
        name: 'RemoteDemo',
        // 格式设置
        formats: ['umd'],
        fileName: (format) => `remote-demo-lib.${format}.js`,
      },
      // 外部依赖声明
      rollupOptions: {
        external: [
          'react',
          'react-dom',
          'react/jsx-runtime',
          'antd',
          'antd/es/*',
        ],
        output: {
          globals: {
            react: 'React',
            'react-dom': 'ReactDOM',
            'react/jsx-runtime': 'React',
            'antd': 'antd',
          },
        },
      },
    },
  }
})

其他

在打包时，可能还需根据业务场景，配置CSS代码、静态资源的打包配置。正常来说，最好是直接都打包到一个完整的js文件中。
