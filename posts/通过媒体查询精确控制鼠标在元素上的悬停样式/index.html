通过媒体查询精确控制鼠标在元素上的悬停样式

2026/1/20 23:59:146 分钟阅读
CSS

在阅读Puck -Button源码时，看到模块样式中使用以下写法：

.Button--primary {
  background: var(--puck-color-azure-04);
}

@media (hover: hover) and (pointer: fine) {
  .Button--primary:hover {
    background-color: var(--puck-color-azure-03);
  }
}

之前没见过这种媒体查询的写法，便网上查询了资料写下这篇文章。

解释

这段CSS代码使用**媒体查询（Media Query）**来实现精细化悬停效果控制。确保悬停颜色变化只会在桌面端鼠标操作时显示，在手机平板上会自动禁用，以提供更合适的用户体验。

Tip

当用户鼠标移入按钮时，按钮颜色从var(--puck-color-azure-04)变为var(--puck-color-azure-03)； 当用户鼠标移出时，按钮颜色恢复为var(--puck-color-azure-04)

鼠标移入按钮时效果：

hover: hover

这里的(hover: hover)分为前后两部分，前一个hover是用于检测设备是否支持悬停效果。后一个hover是两个可选值中的一种：

  • hover: 表示支持悬停效果，如台式机鼠标，触控板
  • none: 表示不支持悬停效果，如手机、平板等触屏设备

即存在下述两种写法：

@media (hover: hover) {
  /* 支持悬停效果时，设置样式 */
}

@media (hover: none) {
  /* 不支持悬停效果时，设置样式 */
}

pointer: fine

这里的(pointer: fine)分为前后两部分，前一个pointer是用于检测指针设备是否精确，后一个fine是三个可选值中的一种：

  • fine: 表示具备鼠标、触控笔等精确指针
  • coarse: 表示具备手指触控等不精确指针
  • none: 表示无指针设备

即存在下述三种写法：

@media (pointer: fine) {
  /* 支持精确指针，设置样式 */
}

@media (pointer: coarse) {
  /* 支持不精确指针，设置样式 */
}

@media (pointer: none) {
  /* 不支持指针，设置样式 */
}

设计意图

这种写法的核心目的是：

  • 设备适配精准：只为真正需要悬停反馈的设备提供效果，避免在触屏设备上产生混乱的视觉反馈
  • 移动端友好：手机上没有不必要的颜色闪烁
  • 辅助功能考虑：对某些特殊输入设备更友好，避免对屏幕阅读器等造成干扰
